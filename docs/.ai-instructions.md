# Instructions pour les IA - VehicleMarket Project

## 🎯 Contexte du projet

**VehicleMarket** est une plateforme de vente de véhicules au Bénin avec gestion des rendez-vous, devis et SAV.

- **Backend**: Node.js + Express + MySQL (Port 5001)
- **Frontend**: Next.js 15 + React 19 + TypeScript (Port 3000)
- **Base de données**: MySQL (Port 3306, nom: `auto`)
- **Devise**: FCFA (Franc CFA) - Position: après le montant

## 📁 Structure du projet

```
auto/
├── backend/              # API Node.js + Express
│   ├── src/
│   │   ├── config/      # Configuration DB
│   │   ├── controllers/ # Logique métier
│   │   ├── routes/      # Routes API
│   │   ├── middleware/  # Auth, validation
```

## 🗄️ Base de données - IMPORTANT

### Commandes disponibles

```bash
# Migration complète (créer toutes les tables)
npm run db:migrate

# Seed complet (insérer toutes les données)
npm run db:seed

# Reset complet (migration + seed)
npm run db:reset

# Vérifier les tables
npm run db:check
```

### Fichiers de base de données

**UNIQUEMENT ces fichiers doivent être utilisés :**

- ✅ `migrate-all.js` - Migration complète (11 tables)
- ✅ `seed-all.js` - Seed complet (toutes les données)
- ✅ `check-tables.js` - Vérification des tables
- ✅ `README.md` - Documentation

**NE PAS utiliser ou créer :**
- ❌ `migrate.js` (supprimé)
- ❌ `seed.js` (supprimé)
- ❌ `schema.sql` (supprimé)
- ❌ `seed-config.js` (supprimé)
- ❌ `seed-hero.js` (supprimé)
- ❌ Tout autre fichier de migration/seed

### Tables de la base de données (11 au total)

1. **brands** - Marques de véhicules (16 marques par défaut)
2. **hero_settings** - Paramètres de la section hero (structure complète)
3. **users** - Utilisateurs admin
4. **vehicles** - Véhicules disponibles
5. **appointments** - Rendez-vous showroom/SAV
6. **quotes** - Demandes de devis
7. **contacts** - Messages de contact
8. **reviews** - Avis clients
9. **services** - Services SAV
10. **stats** - Statistiques du site
11. **site_config** - Configuration (devise FCFA)

## 📝 Conventions de code

### Backend
- ES Modules (`import/export`)
- Async/await pour les requêtes DB
- Validation avec `express-validator`
- Upload avec `multer`

### Frontend
- TypeScript strict
- "use client" pour les composants interactifs
- Tailwind CSS pour le styling
- Composants dans `src/components/`

## ⚠️ Points d'attention

1. **Migrations**: Toujours utiliser `migrate-all.js` (fichier unique)
2. **Seeds**: Toujours utiliser `seed-all.js` (fichier unique)
3. **Ports**: Backend 5001, Frontend 3000
4. **Images**: Servies depuis `backend/uploads/` avec CORS
5. **Devise**: FCFA uniquement, position "after"
6. **Hero**: Structure complète avec 12 champs
7. **API**: Exports nommés, pas de default export

## 🚀 Démarrage rapide

```bash
# Backend
cd backend
npm install
npm run db:reset    # Créer tables + insérer données
npm run dev         # Démarrer serveur (port 5001)

# Frontend
cd frontend
npm install
npm run dev         # Démarrer app (port 3000)
```
