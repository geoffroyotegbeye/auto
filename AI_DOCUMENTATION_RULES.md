# Règles de Documentation pour l'IA

## 📁 Emplacement des Fichiers de Documentation

**IMPORTANT**: Tous les fichiers de documentation markdown (`.md`) doivent être créés dans le dossier `docs/` à la racine du projet.

## 📝 Types de Documentation

### Documentation Technique
- Architecture du projet
- Guides de démarrage
- Instructions de configuration
- Résumés de modifications

### Documentation Utilisateur
- Guides d'utilisation
- Tutoriels
- FAQ
- Troubleshooting

## ✅ Règles à Suivre

1. **Emplacement obligatoire**: `docs/` pour tous les fichiers `.md`
2. **Nommage**: Utiliser des noms descriptifs en MAJUSCULES avec underscores
   - Exemple: `GUIDE_DEMARRAGE.md`, `ARCHITECTURE.md`
3. **Langue**: Français pour ce projet
4. **Format**: Markdown standard avec syntaxe GitHub Flavored Markdown

## 🚫 Ne PAS Faire

- ❌ Ne pas créer de fichiers `.md` à la racine du projet
- ❌ Ne pas dupliquer la documentation existante
- ❌ Ne pas créer de documentation inutile ou redondante

## 📂 Structure du Dossier docs/

```
docs/
├── ARCHITECTURE.md          # Architecture technique du projet
├── GUIDE_DEMARRAGE.md       # Guide de démarrage rapide
├── QUICK_START.md           # Démarrage ultra-rapide
├── CHANGELOG.md             # Historique des modifications
├── TROUBLESHOOTING.md       # Résolution de problèmes
├── INSTRUCTIONS_BDD.md      # Instructions base de données
├── PORTS_CONFIG.md          # Configuration des ports
└── ...                      # Autres fichiers de documentation
```

## 💡 Bonnes Pratiques

- Mettre à jour la documentation existante plutôt que d'en créer une nouvelle
- Utiliser des titres clairs et une structure hiérarchique
- Inclure des exemples de code quand c'est pertinent
- Ajouter des emojis pour améliorer la lisibilité
- Garder la documentation concise et à jour

---

**Note**: Cette règle s'applique à tous les assistants IA travaillant sur ce projet.
